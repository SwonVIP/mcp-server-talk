Getting Started with FluxJS

FluxJS is a lightweight, modern JavaScript framework for building reactive user interfaces. With its intuitive component-based architecture and powerful state management, FluxJS makes it easy to create scalable web applications.

What is FluxJS?

FluxJS is a progressive framework that focuses on simplicity and performance. It provides a declarative approach to building UIs, making your code more predictable and easier to debug.

Installation

You can install FluxJS using npm, yarn, or include it directly via CDN:

1 Install via npm
npm install fluxjs
2 Install via yarn
yarn add fluxjs
3 Include via CDN
<script src="https://cdn.jsdelivr.net/npm/fluxjs@2.1.0/dist/flux.min.js"></script>

Quick Start

Here's a simple example to get you started with FluxJS. This creates a basic counter component:

import { Component, createElement } from 'fluxjs'; class Counter extends Component { constructor() { super(); this.state = { count: 0 }; } increment() { this.setState({ count: this.state.count + 1 }); } render() { return createElement('div', { children: [ createElement('h2', { text: `Count: ${this.state.count}` }), createElement('button', { text: 'Increment', onClick: () => this.increment() }) ] }); } } // Mount the component const app = new Counter(); app.mount('#app');
Pro Tip: FluxJS components automatically re-render when their state changes, ensuring your UI stays in sync with your data.

Project Setup

Follow these step-by-step instructions to create a new FluxJS project from scratch:

1 Create the main project folder

Create a new folder called "FluxMain" for your project:

mkdir FluxMain cd FluxMain
2 Create component files

Create the main component HTML and SCSS files:

touch component.html component.scss

The component.html file will contain your component templates, and component.scss will contain your styles.

3 Set up routing configuration

Create a routing configuration file:

touch routing.json

Example routing.json structure:

{ "routes": [ { "path": "/", "component": "HomeComponent", "name": "home" }, { "path": "/about", "component": "AboutComponent", "name": "about" }, { "path": "/contact", "component": "ContactComponent", "name": "contact" } ], "defaultRoute": "/", "hashRouting": true }
4 Configure deployment server

Create a deployment configuration file:

touch deploy.json

Example deploy.json configuration:

{ "server": { "host": "localhost", "port": 3000, "staticFiles": "./dist", "publicPath": "/public" }, "build": { "outputDir": "./dist", "minify": true, "sourceMaps": true }, "deployment": { "target": "production", "environment": { "NODE_ENV": "production", "API_URL": "https://api.yourapp.com" } } }
Project Structure: Your FluxMain folder should now contain: component.html, component.scss, routing.json, and deploy.json

Complete Project Structure

After following the setup steps, your project structure should look like this:

FluxMain/ ├── component.html # Component templates ├── component.scss # Component styles ├── routing.json # Route configuration ├── deploy.json # Deployment settings └── package.json # Dependencies (after npm init)

Next: Initialize Your First Component

Now you can start building your application. Here's how to create your first component in component.html:

<!-- component.html --> <template id="home-component"> <div class="home-container"> <h1>Welcome to FluxJS</h1> <p>Your first FluxJS application is ready!</p> <button onclick="this.handleClick()">Get Started</button> </div> </template> <template id="about-component"> <div class="about-container"> <h1>About FluxJS</h1> <p>Learn more about this amazing framework.</p> </div> </template>

And add corresponding styles in component.scss:

// component.scss .home-container, .about-container { max-width: 600px; margin: 0 auto; padding: 2rem; text-align: center; h1 { color: #6f42c1; margin-bottom: 1rem; } button { background: #6f42c1; color: white; border: none; padding: 0.75rem 1.5rem; border-radius: 0.375rem; cursor: pointer; font-size: 1rem; &:hover { background: #5a339a; } } }

Component Lifecycle

FluxJS components follow a predictable lifecycle that allows you to hook into different stages of a component's existence:

Example Lifecycle Implementation

class UserProfile extends Component { constructor() { super(); this.state = { user: null, loading: true }; } async componentDidMount() { // Fetch user data when component mounts try { const userData = await fetchUser(this.props.userId); this.setState({ user: userData, loading: false }); } catch (error) { this.setState({ error: error.message, loading: false }); } } componentDidUpdate(prevState) { // Log state changes for debugging if (prevState.user !== this.state.user) { console.log('User data updated:', this.state.user); } } componentWillUnmount() { // Clean up any subscriptions or timers if (this.subscription) { this.subscription.unsubscribe(); } } render() { if (this.state.loading) { return createElement('div', { text: 'Loading...' }); } return createElement('div', { children: [ createElement('h2', { text: this.state.user.name }), createElement('p', { text: this.state.user.email }) ] }); } }
Note: Always clean up subscriptions, timers, and event listeners in componentWillUnmount() to prevent memory leaks.
